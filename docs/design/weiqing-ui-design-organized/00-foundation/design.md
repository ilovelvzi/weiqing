# DESIGN.md — 微轻 App 设计系统与页面设计规范

**产品名称**：微轻 App  
**文档类型**：Design System / Page Design Specification  
**版本**：V1.0  
**适用平台**：iOS 优先  
**设计基线**：Apple Human Interface Guidelines（HIG）/ iOS 原生体验 / HealthKit 合规  
**视觉方向**：苹果灰白 + 蓝色点缀 / 轻盈 / 克制 / 治愈 / 陪伴感  
**更新时间**：2026-05-24

---

## 0. 文档目标

本文件是「微轻 App」的统一 UI 设计规范，用于指导设计稿、前端实现、AI 设计稿生成、组件拆分与验收。

它的目标不是单纯让页面“好看”，而是让产品满足以下标准：

1. 符合 Apple Human Interface Guidelines；
2. 符合 iOS 用户的操作习惯；
3. 保护体重、健康、饮食、AI 上下文等敏感数据；
4. 保持核心打卡链路轻量、低压、可持续；
5. 降低研发落地成本和 App Store 审核风险；
6. 形成可长期扩展的设计系统。

## 1. 产品设计原则

### 1.1 Clarity 清晰

每个页面只突出一个主任务。

| 页面 | 主任务 |
|---|---|
| 首页 | 让用户快速看到今日状态并记录体重 |
| 打卡页 | 让用户快速输入一次体重 |
| 记录页 | 查看、补录、编辑、删除历史记录 |
| 分析页 | 看见趋势、目标进度和健康指标 |
| 我的页 | 管理资料、同步、隐私、安全和设置 |
| 周报/月报 | 给用户阶段性反馈与分享素材 |
| 数据导出页 | 安全导出个人数据 |
| 注销页 | 清晰完成高风险账号操作 |

设计要求：

- 标题、数字、按钮、输入状态必须一眼可理解；
- 不用复杂术语；
- 不在首页堆叠过多健康指标；
- 不用焦虑型文案刺激用户。

### 1.2 Deference 尊重内容

微轻的核心内容是：体重数字、趋势曲线、目标进度、AI 鼓励、健康数据和用户自己的记录。

UI 必须服务内容，而不是抢夺注意力。

设计要求：

- 卡片背景低饱和；
- 文字对比清晰；
- 装饰性渐变不得影响阅读；
- 图表清楚但不刺眼；
- 不用大面积插画压过核心数字。

### 1.3 Depth 层次感

层次感用于帮助用户理解结构，而不是为了炫技。

Liquid Glass / 毛玻璃类视觉仅允许用于：

- Tab Bar；
- Navigation Bar；
- Toolbar；
- 底部浮动操作区。

禁止用于：

- 今日体重主卡片；
- 趋势图内容卡片；
- AI 鼓励卡片；
- 健康指标卡片；
- 表单输入区；
- 注销、导出等高风险说明区。

内容卡片使用 `systemBackground` / `secondarySystemBackground` / 轻阴影 / 大圆角 / 清晰文字层级。

## 2. HIG 合规硬性约束

### 2.1 导航

底部 Tab 固定为 4 个：

1. 首页；
2. 记录；
3. 分析；
4. 我的。

规则：

- Tab Bar 只用于一级导航；
- 打卡不是 Tab，不放在 Tab 中间做凸起按钮；
- 打卡动作放在首页主卡片和底部主按钮；
- 社区不进入主 Tab，仅作为附录可裁剪模块；
- 导航层级尽量不超过 3 层。

### 2.2 触控区域

所有可点击元素触控热区不小于 **44×44pt**。

适用对象：

- Tab；
- 返回 / 关闭按钮；
- 体重单位切换；
- 标签 Chip；
- 日期选择；
- 图表点位；
- 删除按钮；
- Toggle；
- 设置列表行；
- 小组件预览项。

### 2.3 Safe Area

所有关键内容必须位于 Safe Area 内：

- 顶部内容不得被刘海 / Dynamic Island 遮挡；
- 底部按钮不得被 Home Indicator 遮挡；
- 底部 Tab Bar 上方内容需留出滚动空间；
- Sheet 和 Alert 避免遮挡关键操作。

### 2.4 字体与 Dynamic Type

使用 Apple 系统字体体系：

| 场景 | 字体建议 |
|---|---|
| 首页体重数字 / 大标题 | SF Pro Display |
| 正文 / 设置项 / 说明文字 | SF Pro Text |

必须支持 Dynamic Type。字号放大后：

- 体重数字不能截断；
- 卡片高度自适应；
- 设置项说明文字可换行；
- 图表摘要可以换行或折叠；
- 底部按钮文字不能重叠。

建议字号：

| 元素 | 建议字号 |
|---|---:|
| 首页体重数字 | 48–64pt |
| 页面大标题 | 28–34pt |
| 卡片标题 | 17–20pt |
| 正文 | 15–17pt |
| 辅助说明 | 13–15pt |
| 按钮 | 17pt semibold |

### 2.5 色彩与对比度

主风格：苹果灰白背景 + 柔和蓝色强调色。

必须支持：

- Light Mode；
- Dark Mode；
- 高对比度；
- 色弱用户可识别。

规则：

- 正文文字对比度不低于 WCAG AA；
- 不能只靠颜色表达状态；
- 不用红色表达“体重上升”；
- 不用绿色暗示“瘦才是成功”；
- 危险操作用红色弱警示，并配合明确文案。

示例：

- `较上次 -0.3kg`：文字 + 温和色彩；
- `较上次 +0.2kg`：中性色 + 温和说明；
- `权限未开启`：图标 + 文案 + 操作按钮。

### 2.6 权限请求

权限请求必须先解释价值，再触发系统权限。

适用：

- 通知权限；
- Apple Health 权限；
- Face ID / Touch ID；
- 相册 / 相机权限（仅未来社区模块启用时）。

Apple Health 页面必须明确：

- 读取 Apple Health 中的体重数据；
- 写入微轻中手动记录的体重数据；
- 读取和写入是两个独立开关；
- 导入由用户手动触发；
- 微轻不会后台自动读取；
- 注销微轻不会删除 Apple Health 中已有数据。

### 2.7 弹窗与 Sheet

Alert 仅用于关键决策：

- 删除记录；
- 注销账号；
- 开启小组件显示体重数字；
- 退出登录且存在未同步数据；
- 数据导出身份验证失败；
- Apple Health 权限异常。

规则：

- Alert 最多两个按钮；
- 多选、多字段设置使用底部 Sheet；
- 日期 + 时间 + 时段选择使用 Sheet；
- 删除、注销、显示敏感数字必须二次确认。

### 2.8 无障碍

必须支持：

- VoiceOver；
- Dynamic Type；
- 深色模式；
- 高对比度；
- 减少动态效果；
- 外接键盘基础操作。

示例 VoiceOver 文案：

| 元素 | 描述 |
|---|---|
| 今日体重主卡片 | 今日体重，65.5 公斤，较上次减少 0.3 公斤 |
| 打卡按钮 | 轻轻记录一下，按钮 |
| 目标进度 | 目标进度，已完成 42% |
| 趋势图 | 最近 7 天体重趋势图，周期变化减少 0.7 公斤 |
| 删除记录 | 删除这条体重记录，按钮 |

图表不能只依赖曲线视觉，必须提供文字摘要。

## 3. 视觉设计系统

### 3.1 色彩 Tokens

> 实现时优先使用 iOS 语义色，以下为设计语义，不强制绑定具体 Hex。

| Token | 用途 | 建议 |
|---|---|---|
| `color.background.primary` | 页面背景 | systemBackground |
| `color.background.secondary` | 卡片背景 | secondarySystemBackground |
| `color.background.elevated` | 浮层 / Sheet | tertiarySystemBackground |
| `color.text.primary` | 主要文字 | label |
| `color.text.secondary` | 次级文字 | secondaryLabel |
| `color.text.tertiary` | 弱说明 | tertiaryLabel |
| `color.brand.primary` | 主强调色 | 柔和系统蓝 |
| `color.brand.light` | 浅蓝背景 | 低饱和浅蓝 |
| `color.success.soft` | 正向状态 | 温和绿色/蓝绿，但需配文案 |
| `color.warning.soft` | 轻警示 | 柔和橙色 |
| `color.danger` | 危险操作 | systemRed |
| `color.separator` | 分割线 | separator |

禁止：

- 大面积高饱和粉色；
- 高饱和红绿对比；
- 纯颜色表达趋势结论。

### 3.2 圆角

| Token | 值 | 用途 |
|---|---:|---|
| `radius.sm` | 8pt | 小标签、轻按钮 |
| `radius.md` | 12pt | 列表项、输入框 |
| `radius.lg` | 16pt | 普通卡片 |
| `radius.xl` | 24pt | 首页主卡片、报告卡片 |
| `radius.pill` | 999pt | 主按钮、Segmented Control、Tag Chip |

### 3.3 间距

采用 8pt 网格：

| Token | 值 |
|---|---:|
| `space.xs` | 4pt |
| `space.sm` | 8pt |
| `space.md` | 16pt |
| `space.lg` | 24pt |
| `space.xl` | 32pt |
| `space.2xl` | 48pt |

页面左右边距建议：

- iPhone：16–20pt；
- iPad：自适应最大内容宽度，避免全屏拉伸。

### 3.4 阴影与层级

阴影必须轻。内容卡片不要做厚重投影。

| 层级 | 用途 |
|---|---|
| Level 0 | 页面背景 |
| Level 1 | 普通卡片 |
| Level 2 | 首页主卡片 / 报告主卡 |
| Level 3 | Sheet / 浮动底部按钮 |

### 3.5 动效

动效用于反馈，不用于装饰。

允许：

- 打卡成功轻微弹入；
- 数字变化轻过渡；
- Sheet 从底部自然滑入；
- 按钮点击轻触感；
- 图表点位轻微高亮。

必须尊重“减少动态效果”。开启后使用淡入淡出或静态变化。

### 3.6 触觉反馈

触觉反馈只在关键节点使用：

| 操作 | 反馈 |
|---|---|
| 打卡成功 | Success haptic |
| 输入非法被拦截 | Light warning haptic，可选 |
| 切换单位 / 标签选中 | Selection haptic |
| 删除确认 | Warning haptic |
| 导出验证成功 | Success haptic |

## 5. 核心组件规范

### 5.1 主按钮 Primary Button

用途：页面主要行动。

样式：

- 胶囊形；
- 高度不低于 50pt；
- 蓝色填充；
- 白色文字；
- 文字使用动词。

示例：

- 轻轻记录一下；
- 打卡；
- 保存；
- 开始导入；
- 验证并导出。

### 5.2 次按钮 Secondary Button

用途：次要行动。

样式：

- 灰底或描边；
- 胶囊形；
- 不抢主按钮视觉。

示例：

- 稍后再说；
- 暂时跳过；
- 再记一次。

### 5.3 危险按钮 Danger Button

用途：删除、注销、移除设备。

规则：

- 使用红色弱警示；
- 必须二次确认；
- 文案明确说明后果；
- 不用大面积红色吓唬用户。

### 5.4 卡片 Card

卡片用于承载一组内容。

要求：

- 大圆角；
- 低饱和背景；
- 内容层级清楚；
- 卡片内不要塞过多内容。

卡片类型：

- 今日体重主卡；
- 目标进度卡；
- 趋势图卡；
- AI 鼓励卡；
- 健康指标卡；
- 报告摘要卡。

### 5.5 Tag Chip

用途：体重标签、饮食备注标签。

样式：

- 胶囊形；
- 选中态蓝色浅底 + 蓝色文字；
- 未选中态灰底；
- 触控区域 ≥ 44pt；
- 支持多选。

默认体重标签：

- 裸重；
- 饭后；
- 运动后；
- 生理期；
- 聚餐；
- 外卖；
- 高盐；
- 熬夜。

饮食标签：

- 聚餐；
- 外卖；
- 奶茶；
- 甜食；
- 夜宵；
- 高盐；
- 饮酒；
- 暴食；
- 清淡饮食；
- 吃得规律；
- 生理期食欲变化。

### 5.6 Segmented Control

用途：

- 单位切换：kg / 斤 / lb；
- 图表周期：7天 / 30天 / 90天 / 全部；
- Apple Health 导入范围快捷项。

要求：使用系统语义样式，不做过度拟物。

### 5.7 自定义数字键盘

用于体重输入。

键位：

```text
1 2 3
4 5 6
7 8 9
. 0 删除
```

规则：

- 不调用系统键盘；
- 不支持粘贴；
- 非法字符直接拦截；
- 0.0 不允许提交；
- 输入合法时按钮可点击；
- 小数精度根据全局设置处理。

### 5.8 图表 Chart

趋势图规则：

- 按当前单位展示；
- 小数位跟随全局精度；
- 一天多条记录取当天最后一次；
- 缺失日期不显示 Data Dot，但视觉连线；
- 0.0 不进入图表；
- Apple Health 导入记录平权参与；
- Tooltip 不展示 Apple Health 来源。

图表必须有文本摘要，便于无障碍与用户快速理解。

### 5.9 Empty State

空状态必须温和、可行动。

示例：

- `记录几天后，这里会看到你的变化曲线`；
- `还没有体重记录，轻轻记录一下吧`；
- `这个时间范围内没有找到可导入的体重数据`。

避免：

- `暂无数据` 这种冷冰冰文案；
- 焦虑型文案。

## 6. 页面设计规范

### 6.1 首页 Home

目标：3 秒内让用户知道今日状态，并能立刻打卡。

结构：

1. 顶部问候：昵称 + 弱展示连续打卡天数；
2. 今日体重主卡片；
3. 目标进度卡 / 设置目标 Banner；
4. 最近 7 天趋势小图；
5. AI 鼓励卡片；
6. 今日饮食备注轻入口；
7. 底部主按钮：轻轻记录一下。

状态：

- 无记录：今日体重展示 0.0 空状态，不进入图表；
- 已打卡：展示今日最新体重、较上次变化、时段；
- 离线：顶部轻提示“当前网络不可用，记录会先保存在本机”；
- PENDING_DELETE：不进入首页。

首页禁止展示：

- BMI 详情；
- BMR；
- 体脂率；
- 靶心率；
- 社区内容；
- 广告位；
- 大段健康科普。

### 6.2 体重打卡输入页

标题：轻轻记录一下。

结构：

1. 顶部关闭 / 标题 / 日期时段入口；
2. 大数字输入区；
3. 单位切换 kg / 斤 / lb；
4. 日期 + 时间 + 时段选择；
5. 标签多选区；
6. 自定义数字键盘；
7. 底部按钮：打卡。

规则：

- 已输入数字后切换单位，自动换算展示；
- 支持补打卡，范围最近 5 年至今天；
- 不允许未来日期时间；
- 支持多选标签；
- 支持新建标签并进入常用标签列表；
- 不支持图片；
- 不支持备注；
- 不支持围度；
- 不展示 Apple Health 来源。

### 6.3 打卡成功反馈页

目标：强化完成感和陪伴感。

内容：

- 记录完成啦；
- 本次体重；
- 较上次变化；
- AI 鼓励文案；
- 回到首页；
- 再记一次。

AI 加载：

- `正在为你生成一句鼓励...`

AI 失败：

- 使用兜底文案，不阻断打卡。

AI 超额：

- 不提示“额度用完”，使用温和本地文案。

### 6.4 记录页 Records

结构：

1. 顶部标题：记录；
2. 右上角：+ 补打卡；
3. 月份切换；
4. 按日期分组列表；
5. 每日饮食备注轻入口。

规则：

- 暂不做日历热力图；
- 一天多条记录按 recorded_at 从晚到早；
- 记录卡片展示体重、时段/时间、较上一条变化、标签；
- `period = none` 时展示 HH:mm；
- 点击记录进入详情编辑；
- 支持左滑删除；
- 删除前二次确认；
- Apple Health 导入记录不展示来源。

### 6.5 记录详情 / 编辑页

允许编辑：

- 体重数值；
- 单位；
- 标签；
- 时段。

禁止编辑：

- 日期；
- 时间；
- 图片；
- 备注；
- 围度；
- source。

日期时间只读展示。

### 6.6 饮食备注页

定位：轻量饮食线索，不做热量、不做食物库。

结构：

1. 日期；
2. 饮食标签；
3. 简短文字；
4. 保存。

规则：

- 标签多选；
- 支持自定义标签；
- 文字最多 100 字；
- 不支持图片；
- 不支持热量；
- 不支持食物库；
- 按 local_date 归属；
- 支持离线创建、编辑、删除。

### 6.7 分析页 Analysis

结构：

1. 周期切换：7天 / 30天 / 90天 / 全部；
2. 体重趋势大图；
3. 周期摘要；
4. 目标进度卡片；
5. 健康指标卡片组：BMI / BMR / 体脂率 / 靶心率；
6. 周报 / 月报入口。

规则：

- 默认 7 天；
- 图表点位点击展示日期、体重、较上次变化、标签；
- 不展示 Apple Health 来源；
- 周期摘要展示周期变化、记录天数、最高、最低；
- 饮食备注不在分析页展开，只在周报/月报体现；
- 不展示社区内容。

### 6.8 我的页 Settings

采用 iOS Grouped List 风格。

分组：

1. 顶部个人资料卡：昵称、连续打卡天数、目标体重；
2. 健康资料：身高 / 出生年月 / 性别 / 目标体重 / 体脂率 / 围度数据；
3. 记录设置：体重单位 / 小数精度 / 常用标签管理 / 饮食备注标签管理；
4. 智能与提醒：AI 鼓励 / 每日提醒 / 周报月报提醒 / 小组件设置；
5. 数据与同步：Apple Health / 数据导出 / 同步说明；
6. 账号与安全：登录方式 / 绑定手机号 / 已登录设备 / 修改密码 / 退出登录 / 注销账号；
7. 关于与协议：隐私政策 / 用户协议 / 健康免责声明 / 版本号 / 反馈与帮助。

### 6.9 新手引导

流程：

1. 启动页；
2. 3 屏以内产品价值引导；
3. 登录页；
4. 基础资料填写；
5. 目标体重引导；
6. 通知权限引导；
7. Apple Health 引导；
8. 进入首页。

启动页文案：

- `轻轻记录，看见变化`

注册必填：

- 身高；
- 出生年月；
- 性别。

目标体重强引导但允许跳过。

### 6.10 Apple Health 页面

入口：我的 → 数据与同步 → Apple Health。

结构：

1. 顶部说明卡；
2. 读取开关；
3. 写入开关；
4. 手动导入区；
5. 最近导入结果；
6. FAQ / 权限说明。

规则：

- 导入由用户手动触发；
- 不做后台自动读取；
- 读取 / 写入独立开关；
- 导入范围：最近 7 天 / 最近 30 天 / 最近 1 年 / 自定义；
- 默认最近 30 天；
- 自定义最大 3 年；
- 导入结果展示成功、重复跳过、已删除跳过、失败数量；
- 不展示 UUID / external_uuid；
- 写入失败不影响本地打卡成功。

### 6.11 数据导出页

入口：我的 → 数据与同步 → 数据导出；PENDING_DELETE 冷静期页也提供入口。

结构：

1. 导出说明；
2. 会导出的数据；
3. 不会导出的敏感内部字段；
4. 身份验证；
5. 导出任务状态；
6. 下载链接。

规则：

- 导出文件为 ZIP，内部使用 CSV；
- 主产品导出不包含图片；
- 导出前必须系统生物识别；
- 无生物识别时用密码 / 验证码降级；
- 下载链接 24 小时有效；
- DELETING / DELETED 状态不可导出；
- 不展示 external_uuid、内部任务 ID、技术错误码。

### 6.12 账号注销流程

入口：我的 → 账号与安全 → 注销账号。

流程：

1. 注销说明页；
2. 风险勾选；
3. 身份强验证；
4. 申请成功页；
5. PENDING_DELETE 冷静期页。

规则：

- 注销入口红色弱警示；
- 风险勾选必须全部勾选；
- 支持生物识别 / 验证码 / 密码；
- PENDING_DELETE 登录后进入冷静期页，不进入首页；
- 冷静期页展示剩余天数、预计删除时间、恢复账号、导出数据；
- 不提供提前立即删除；
- 不提供继续注销按钮，只说明到期自动删除；
- DELETING / DELETED 不允许恢复和导出；
- 明确说明不会删除 Apple Health 中已有数据。

### 6.13 设备管理页

入口：我的 → 账号与安全 → 已登录设备。

结构：

1. 顶部说明最多 5 台设备；
2. 当前设备卡片；
3. 其他设备列表；
4. 安全说明。

展示字段：

- 设备名称；
- 最后活跃时间；
- 登录城市。

不展示：

- 系统版本；
- App 版本；
- 登录 IP；
- Token。

规则：

- 可移除其他设备；
- 移除前二次确认；
- 可移除当前设备，等同更彻底退出登录；
- 被移除设备本地数据冻结不删除；
- 重新登录同一账号后可继续同步未同步数据；
- 离线时不展示远程设备列表。

### 6.14 提醒设置页

入口：我的 → 智能与提醒 → 提醒设置。

规则：

- 合并每日提醒、周报、月报；
- 展示系统通知权限状态；
- 未授权时引导去系统设置；
- 每日体重提醒默认开启；
- 默认提醒时间 08:00；
- 最多 3 个提醒时间；
- 不允许重复时间；
- 关闭每日提醒后，时间保留但置灰；
- 当天已打卡后，仍保留剩余提醒；
- 未打卡唤醒：第 2 天 / 第 4 天 / 第 6 天，之后静默；
- 周报通知默认开启；
- 月报通知默认开启；
- PENDING_DELETE 后停止全部提醒和报告推送。

### 6.15 AI 鼓励设置页

入口：我的 → 智能与提醒 → AI 鼓励。

规则：

- 默认开启；
- 关闭后停止生成并停止向 AI 服务发送上下文；
- 风格：温柔陪伴 / 元气打气 / 理性分析 / 极简一句话；
- 默认风格：温柔陪伴；
- 展示今日剩余次数，例如 7 / 10；
- 说明 AI 会参考哪些信息；
- 说明不会发送手机号、邮箱、设备 Token、Apple Health UUID、精确位置；
- 展示健康安全免责声明。

### 6.16 小组件设置页

入口：我的 → 智能与提醒 → 小组件设置。

规则：

- 默认不展示具体体重数字；
- 开启显示体重数字前二次确认；
- 小组件只作为快捷入口，不直接输入体重；
- 第一版支持小号 / 中号 / 锁屏三种基础样式；
- 展示隐私模式和显示数字模式预览。

### 6.17 目标体重设置页

入口：

- 首页目标 Banner；
- 分析页目标卡片；
- 我的 → 健康资料 → 目标体重。

规则：

- 目标体重单位跟随当前单位；
- 合法范围与体重输入一致；
- 系统自动判断减重 / 增重；
- 初始体重默认取设置目标时最近一次有效体重；
- 用户可手动覆盖初始体重；
- 修改目标后重新计算进度；
- 历史报告快照不自动改变。

### 6.18 健康数据页

入口：我的 → 健康资料；分析页健康指标卡片。

包含：

- 身高；
- 出生年月；
- 性别；
- BMI；
- BMR；
- 体脂率；
- 靶心率；
- 围度。

规则：

- 围度不在打卡页填写；
- 围度单位 cm；
- 围度范围 10–300cm；
- 体脂率范围 3%–70%；
- 修改基础资料后 BMI / BMR / 靶心率立即重算；
- 历史报告快照不自动改变。

### 6.19 周报 / 月报页

入口：分析页底部。

周报包含：

- 成果摘要；
- 趋势图；
- 饮食线索；
- 健康数据摘要；
- AI 鼓励。

月报包含：

- 记录天数；
- 体重变化；
- 最高 / 最低体重；
- 目标进度；
- 趋势图；
- AI 总结。

规则：

- 报告保存为快照；
- 后续历史数据编辑不自动改变已生成报告；
- 暂不支持手动重新生成；
- 数据不足时仍允许生成，但弱化趋势变化；
- 体重上升时使用温和表达；
- 不做饮食因果判断，只做线索提示；
- PENDING_DELETE 停止生成新报告。

### 6.20 分享图生成页

规则：

- 默认隐藏绝对体重数字；
- 默认不显示头像和昵称；
- 用户可手动开启显示昵称；
- 用户可手动开启显示头像；
- 开启显示具体数字前二次确认；
- 第一版只做 9:16 竖图；
- 必须带微轻 Logo / 水印。

## 7. 隐私与安全 UX

### 7.1 体重数字隐私

默认隐藏敏感数字的场景：

- 分享图；
- 小组件；
- 锁屏小组件；
- 注销冷静期页面；
- 未登录 / 退出登录后。

用户主动开启显示敏感数字前必须二次确认。

### 7.2 退出登录

如果存在未同步数据，退出登录前必须拦截：

```text
你有 X 条体重数据尚未同步到云端，此时退出可能导致多端数据不一致。是否等待同步完成后退出？
```

按钮：

- 尝试同步并退出；
- 强行退出。

### 7.3 PENDING_DELETE

进入 PENDING_DELETE 后：

- 不进入首页；
- 不进入记录页；
- 不进入分析页；
- 不允许新增、编辑、删除体重记录；
- 允许导出数据；
- 允许恢复账号；
- 停止全部通知和报告推送。

## 8. AI 设计规则

AI 鼓励定位：即时反馈和陪伴，不是减肥教练、医生、营养师。

允许：

- 使用用户昵称；
- 提到具体体重数字；
- 温和提及生理期；
- 温和提及暴食，但不得羞辱或诊断。

禁止：

- 医疗诊断；
- 处方建议；
- 极端节食；
- 断食、催吐、泻药、利尿剂、减肥药建议；
- 体型羞辱；
- 制造体重焦虑；
- 承诺医学效果。

检测到“吃药、减肥药、利尿剂”等风险词时：

- 屏蔽该词；
- 不评价药物；
- 不给药物建议；
- 输出安全、泛化、正向鼓励。

## 9. AI 设计稿生成全局提示词

所有页面提示词都必须追加以下全局约束：

```text
遵循 Apple Human Interface Guidelines，使用 iOS 原生设计语言。支持 Safe Area、Dynamic Type、VoiceOver、Light Mode 和 Dark Mode，所有触控区域不小于 44pt。使用 SF Pro 字体和 SF Symbols。底部 Tab Bar 与 Navigation Bar 可使用轻量 Liquid Glass / 系统材质，但内容卡片不要使用玻璃材质。整体视觉为苹果灰白背景 + 柔和蓝色强调色，界面清晰、克制、留白充足、低焦虑、治愈陪伴感。不要展示社区内容、广告或医疗诊断建议。
```

## 10. 设计验收清单

### 10.1 HIG 与平台

- [ ] 底部 Tab 不超过 5 个；
- [ ] 关键内容位于 Safe Area 内；
- [ ] 所有触控区域 ≥ 44×44pt；
- [ ] 使用 iOS 原生导航、列表、Sheet、Alert 语言；
- [ ] Liquid Glass 只用于导航层；
- [ ] 支持 Light / Dark Mode；
- [ ] 支持 Dynamic Type；
- [ ] 支持 VoiceOver；
- [ ] 尊重减少动态效果。

### 10.2 核心体验

- [ ] 首页 3 秒内能看到今日状态和打卡入口；
- [ ] 打卡页不包含图片、备注、围度；
- [ ] 打卡成功有轻量反馈和 AI 鼓励；
- [ ] 记录页支持补打卡、编辑、删除；
- [ ] 分析页包含趋势、摘要、目标和健康指标；
- [ ] 我的页采用 iOS 分组设置风格；
- [ ] 数据导出、注销、设备管理流程清晰。

### 10.3 隐私安全

- [ ] 退出登录后离线也不可查看上次账号数据；
- [ ] 分享图默认隐藏绝对体重；
- [ ] 小组件默认隐藏具体体重；
- [ ] Apple Health 读写权限分开说明；
- [ ] 数据导出前强验证；
- [ ] 注销流程有风险勾选和强验证；
- [ ] PENDING_DELETE 停止全部提醒和报告推送。

### 10.4 健康合规

- [ ] BMI / BMR / 靶心率展示免责声明；
- [ ] AI 不提供医疗、药物、极端减肥建议；
- [ ] 饮食备注不做热量和营养处方；
- [ ] 体脂率仅用户手动填写，不自动估算；
- [ ] 分享与报告不制造体重焦虑。
