# 微轻 App 页面设计说明与 AI 设计稿提示词 V1.0

**产品名称**：微轻 App  
**文档类型**：页面设计说明 / AI 设计稿提示词  
**版本**：V1.0  
**适用平台**：iOS 优先  
**设计基线**：Apple Human Interface Guidelines（HIG）/ iOS 原生体验 / HealthKit 合规  
**主视觉方向**：苹果灰白 + 蓝色点缀 / 轻盈 / 克制 / 治愈 / 陪伴感  
**生成时间**：2026-05-24

---

## 0. 文档目标

本文档用于指导「微轻 App」从 PRD 进入页面设计阶段，输出可直接用于 Stitch、Figma AI、Claude Artifacts、Uizard、Galileo、v0 等 AI 设计工具的页面提示词。

设计目标不是单纯“好看”，而是：

1. 符合 Apple Human Interface Guidelines；
2. 符合 iOS 用户习惯；
3. 满足微轻已确认的 PRD 功能边界；
4. 降低研发落地成本；
5. 降低 App Store 审核风险；
6. 保持体重打卡产品的轻量、隐私、安全与陪伴感。

---

## 1. Apple HIG 合规设计约束

### 1.1 核心原则

#### Clarity 清晰

- 页面必须干净、精确、低噪音。
- 一个页面只突出一个主任务。
- 按钮、标题、输入、结果都必须一眼可理解。
- 禁止为了视觉氛围牺牲可读性。

| 页面 | 主任务 |
|---|---|
| 首页 | 让用户快速记录体重 |
| 打卡页 | 让用户输入一次体重 |
| 记录页 | 查看和管理历史记录 |
| 分析页 | 看见趋势与目标进度 |
| 我的页 | 管理资料、同步、隐私与设置 |
| 周报/月报 | 给用户阶段性反馈 |
| 数据导出页 | 安全导出个人数据 |
| 注销页 | 清晰完成高风险账号操作 |

#### Deference 尊重内容

微轻的核心内容是体重数字、趋势曲线、目标进度、AI 鼓励和用户自己的健康记录。UI 应服务于内容，而不是抢内容。

设计要求：

- 卡片背景低饱和；
- 装饰性渐变不能影响文字；
- 图表线条清楚但不刺眼；
- 不用大面积插画压过核心数字；
- 不在首页堆叠过多指标。

#### Depth 层次感

可以使用层次，但要克制。

Liquid Glass / 毛玻璃类视觉只允许出现在：

- Tab Bar；
- Navigation Bar；
- Toolbar；
- 底部浮动操作区。

禁止用于：

- 体重主卡片；
- 趋势图内容卡片；
- AI 鼓励卡片；
- 健康指标卡片；
- 表单输入区；
- 数据导出、注销等高风险说明区。

内容卡片采用 systemBackground / secondarySystemBackground、轻阴影、大圆角和清晰文字层级。

---

### 1.2 平台与布局约束

设计稿必须考虑 iPhone SE、标准 iPhone、Pro Max、iPad 竖屏和 iPad 横屏基础适配。

所有关键内容必须在 Safe Area 内：

- 顶部内容不得被刘海 / Dynamic Island 遮挡；
- 底部按钮不得被 Home Indicator 遮挡；
- 底部 Tab Bar 上方内容需留出足够滚动空间；
- 弹窗和 Sheet 必须避开安全区域。

所有可点击元素触控区域不小于 **44×44pt**。

适用对象包括：Tab、返回按钮、关闭按钮、单位切换、标签 Chip、日期选择、图表点位、删除按钮、开关、行列表项。

---

### 1.3 字体与排版

使用 Apple 系统字体体系：

- 大数字、展示级标题：SF Pro Display；
- 正文、说明、设置项：SF Pro Text。

必须支持 Dynamic Type：用户放大字体后，按钮文字不能截断，卡片高度应自适应，设置项说明文字可换行，图表摘要应可换行或折叠。

字号建议：

| 元素 | 建议 |
|---|---|
| 首页体重数字 | 48–64pt |
| 页面标题 | 28–34pt |
| 卡片标题 | 17–20pt |
| 正文 | 15–17pt |
| 辅助说明 | 13–15pt |
| 按钮 | 17pt semibold |

---

### 1.4 色彩规范

主色方向：苹果灰白背景 + 柔和蓝色强调色。

必须支持：

- Light Mode；
- Dark Mode；
- 高对比度；
- 色弱用户可识别。

禁止：

- 只用颜色传达状态；
- 用红色表达“体重上升”；
- 用绿色强暗示“瘦才是成功”；
- 大面积粉色幼态设计；
- 背景过度渐变影响文字对比度。

状态表达必须同时包含文字，例如“较上次 -0.3kg”“较上次 +0.2kg”“数据不足”“未开启权限”。

---

### 1.5 导航规范

底部 Tab 固定为 4 个：

1. 首页；
2. 记录；
3. 分析；
4. 我的。

规则：

- Tab Bar 只用于一级导航；
- 打卡不是 Tab，不放在 Tab 中间作为凸起按钮；
- 打卡动作放在首页主卡片和底部主按钮；
- Tab 不超过 5 个；
- 社区不进入主 Tab，作为附录可裁剪模块。

导航层级控制：一级 Tab，二级功能页，三级详情 / 编辑 / 强确认页，尽量不超过 3 层。

---

### 1.6 系统组件优先

设计稿优先采用 iOS 原生组件语言：Navigation Bar、Tab Bar、Grouped List、Sheet、Alert、Action Sheet、Segmented Control、Toggle、Date Picker、Picker、SF Symbols、Large Title、Capsule Button。

设置类页面必须接近 iOS 系统设置风格，不做过度装饰。

---

### 1.7 弹窗与 Sheet 规范

Alert 只用于关键决策：删除记录、注销账号、导出身份验证失败、开启小组件显示体重数字、退出登录且存在未同步数据、Apple Health 权限异常。

Alert 最多两个按钮：取消 / 确认操作。

多选、多字段设置使用底部 Sheet，例如日期 + 时间 + 时段选择、导入范围选择、分享图隐私选项、AI 风格选择。

---

### 1.8 权限请求规范

权限请求必须先解释价值，再触发系统权限。

适用：通知权限、Apple Health 权限、Face ID / Touch ID、相册 / 相机权限（仅未来社区模块启用时）。

Apple Health 页面必须明确：

- 读取体重数据；
- 写入微轻手动记录；
- 读取和写入是两个独立权限；
- 导入由用户手动触发；
- 微轻不会后台自动读取；
- 注销微轻不会删除 Apple Health 中已有数据。

---

### 1.9 无障碍规范

必须支持 VoiceOver、Dynamic Type、深色模式、减少动态效果、高对比度和外接键盘基础操作。

示例无障碍文案：

| 元素 | VoiceOver 描述 |
|---|---|
| 今日体重主卡片 | 今日体重，65.5 公斤，较上次减少 0.3 公斤 |
| 打卡按钮 | 轻轻记录一下，按钮 |
| 目标进度 | 目标进度，已完成 42% |
| 趋势图 | 最近 7 天体重趋势图，周期变化减少 0.7 公斤 |
| 删除记录 | 删除这条体重记录，按钮 |

图表不能只依赖视觉曲线，必须提供文字摘要。

---

## 3. 全局视觉风格

### 3.1 关键词

极简、治愈、苹果风、灰白、蓝色点缀、大数字、低压感、女性友好但不过度粉色、陪伴感、空气感、轻健康。

### 3.2 色彩建议

| 用途 | 建议 |
|---|---|
| 背景 | `systemGroupedBackground` |
| 卡片 | `secondarySystemGroupedBackground` / white |
| 强调色 | systemBlue / 柔和蓝 |
| 主文字 | label |
| 辅助文字 | secondaryLabel |
| 分割线 | separator |
| 危险操作 | systemRed |
| 成功反馈 | 蓝色或中性，不强烈绿色化 |

### 3.3 形状

- 主按钮：胶囊形；
- 卡片：20–28pt 圆角；
- 标签 Chip：胶囊形；
- 切换控件：Segmented Control；
- Sheet：大圆角顶部；
- Tab / Nav：可使用 Liquid Glass。

### 3.4 动效

动效只用于页面过渡、打卡成功反馈、图表点位选中、按钮点击、Sheet 出现。必须尊重「减少动态效果」系统设置，禁止纯装饰性复杂动画。

## 5. 页面设计说明与 AI 提示词

---

### 5.1 首页

#### 页面目标

让用户 3 秒内知道：今天有没有记录体重、当前最新体重是多少、距离目标还有多远、可以立即打卡。

#### 页面结构

1. 顶部昵称 + 连续打卡天数弱展示；
2. 今日体重主卡片；
3. 目标进度 / 设置目标 Banner；
4. 最近 7 天趋势小卡片；
5. AI 鼓励卡片；
6. 今日饮食备注轻入口；
7. 底部主按钮“轻轻记录一下”。

#### 关键规则

- 体重数字是视觉中心；
- 今天无记录时显示 0.0 空状态，但 0.0 不进入图表和统计；
- 已打卡后按钮文案可变为“再记一次”；
- 首页不展示 BMI / BMR / 体脂率 / 靶心率详情；
- 首页不展示社区内容；
- 首页不展示 Apple Health 来源。

#### 首页 AI 设计稿提示词

```text
设计一张 iOS 体重打卡 App「微轻」首页界面，风格遵循 Apple Human Interface Guidelines，苹果灰白背景 + 柔和蓝色强调色，治愈、轻盈、克制、留白充足。底部 Tab 为：首页、记录、分析、我的。首页顶部显示昵称问候“早上好，林雨辉”，右侧为头像入口，下方弱展示“已连续记录 12 天”。页面核心是一张大圆角今日体重主卡片，显示“今日体重 65.5 kg”“较上次 -0.3 kg”“今天 · 晚上”，卡片内有小型胶囊按钮“再记一次”。下方展示目标进度卡片：“目标 58.0 kg，还差 3.2 kg，已完成 42%”，使用细进度条。再下方是最近 7 天趋势小图，简洁折线图，有数据点，无数据日期不显示点。然后是 AI 鼓励卡片，温柔文案：“今天已经完成记录啦。数字只是参考，持续看见自己才是最重要的。”底部有固定胶囊主按钮“轻轻记录一下”。使用 SF Pro 字体、SF Symbols、系统卡片、44pt 以上触控区域，支持 Dynamic Type、VoiceOver、Safe Area、Light/Dark Mode。不要展示社区、广告、BMI 详情、热量、食物库。
```

---

### 5.2 体重打卡输入页

#### 页面目标

让用户快速、低压力地完成一次体重输入。

#### 页面结构

1. 顶部关闭按钮 + 标题“轻轻记录一下”；
2. 日期与时段：“今天 · 晚上”；
3. 大数字输入区；
4. 单位切换：kg / 斤 / lb；
5. 标签区；
6. 自定义数字键盘；
7. 底部打卡按钮。

#### 关键规则

- 支持单位切换；
- 已输入数字后切换单位，自动换算展示；
- 点击日期时段进入日期 + 时间 + 时段选择 Sheet；
- 补打卡范围为最近 5 年至今天，不允许未来时间；
- 标签支持多选；
- 支持在打卡页新建自定义标签；
- 不支持图片；
- 不支持备注；
- 不支持围度；
- 不展示 Apple Health 来源。

#### 默认标签

裸重、饭后、运动后、生理期、聚餐、外卖、高盐、熬夜。

#### 打卡页 AI 设计稿提示词

```text
设计一张 iOS App「微轻」体重打卡输入页，标题为“轻轻记录一下”，顶部左侧为关闭按钮，整体遵循 Apple HIG。页面背景为苹果灰白，内容居中。中间大号数字输入区显示“65.5”，右侧单位“kg”，字体使用 SF Pro Display，大数字清晰醒目。数字下方有 Segmented Control：kg / 斤 / lb。再下方是日期时段按钮“今天 · 晚上”，点击感明确，使用胶囊形轻按钮。标签区使用圆角 Chip，多选状态，展示：裸重、饭后、运动后、生理期、聚餐、外卖、高盐、熬夜，并有“+ 自定义”。底部是自定义数字键盘，布局为 1-9、点号、0、删除，键位触控区域不小于 44pt。最底部为蓝色胶囊主按钮“打卡”。界面不出现图片、备注、围度、热量、食物库、Apple Health 来源。支持 Safe Area、Dynamic Type、VoiceOver、Light/Dark Mode。
```

### 5.3 打卡成功反馈页

#### 页面目标

强化“记录完成 + 被鼓励”的即时爽点。

#### 页面结构

1. 成功图标；
2. 标题“记录完成啦”；
3. 本次体重；
4. 较上次变化；
5. AI 鼓励卡片；
6. 按钮：回到首页 / 再记一次。

#### 关键状态

- AI 生成中：显示“正在为你生成一句鼓励…”；
- AI 失败：显示固定兜底文案；
- AI 超过每日次数：显示本地温和文案，不提示“额度用完”。

#### 成功页 AI 设计稿提示词

```text
设计一张 iOS App「微轻」打卡成功反馈页，遵循 Apple HIG，苹果灰白背景，柔和蓝色点缀。页面顶部有轻量成功图标或 SF Symbol checkmark.circle.fill，标题“记录完成啦”。中间展示本次体重“65.5 kg”，下方显示“较上次 -0.3 kg”。下方是一张 AI 鼓励卡片，文案为：“今天的记录很棒。数字只是身体状态的一部分，你愿意持续看见自己，已经是在认真照顾自己了。”底部两个胶囊按钮：主按钮“回到首页”，次按钮“再记一次”。页面清爽、留白充足、无焦虑文案。支持 Dynamic Type、VoiceOver、Safe Area、Light/Dark Mode，触控区域不小于 44pt。
```

### 5.4 记录页

#### 页面目标

查看、编辑、删除历史体重记录，管理轻量饮食备注。

#### 页面结构

1. 顶部标题“记录”；
2. 右上角“+ 补打卡”；
3. 月份切换；
4. 按日期分组列表；
5. 每日饮食备注轻入口。

#### 关键规则

- 暂不做日历热力图；
- 一天多条记录按 recorded_at 从晚到早展示；
- 记录卡片展示体重、时段 / 时间、较上一条变化、标签；
- `period = none` 时展示 HH:mm；
- 点击进入详情编辑；
- 支持左滑删除；
- 删除前二次确认；
- 编辑可改体重、单位、标签、时段；
- 禁止修改日期和时间；
- Apple Health 记录不展示来源；
- 饮食备注只支持标签 + 100 字文字。

#### 记录页 AI 设计稿提示词

```text
设计一张 iOS App「微轻」记录页，使用 iOS Grouped List 与卡片结合的风格，遵循 Apple HIG。顶部大标题“记录”，右上角为“+ 补打卡”按钮。标题下方是月份切换控件“2026年5月”，左右切换箭头。主体为按日期分组的体重记录列表，例如“今天 5月20日”，下面有两张记录卡片：第一条“65.5 kg，今天 · 晚上，较上次 -0.3 kg，标签：裸重、运动后”；第二条“65.8 kg，今天 · 早上，标签：裸重”。日期分组下方有轻入口“今日饮食备注：外卖、奶茶”或“+ 添加饮食备注”。卡片支持点击进入编辑，视觉上暗示可左滑删除。不要展示 Apple Health 来源、图片、备注、围度、热量。界面清晰、列表可读、触控区域大于 44pt，支持 Dynamic Type、VoiceOver、Light/Dark Mode。
```

### 5.5 记录详情 / 编辑页

#### 页面目标

编辑体重数值、单位、标签、时段，删除记录。

#### 页面结构

1. 标题“编辑记录”；
2. 体重输入；
3. 单位切换；
4. 日期时间只读；
5. 时段可修改；
6. 标签可修改；
7. 保存；
8. 删除记录。

#### 关键规则

## 7. 组件规范

### 7.1 主按钮

- 胶囊形；
- 高度 ≥ 50pt；
- 蓝色填充；
- 白色文字；
- 标签使用动词。

示例：轻轻记录一下、打卡、保存、验证并导出、开始导入。

### 7.2 次按钮

- 灰底或边框；
- 胶囊形；
- 不与主按钮抢视觉。

### 7.3 危险按钮

- 红色文字；
- 弱红背景可选；
- 必须二次确认。

示例：删除这条记录、注销账号、移除设备。

### 7.4 标签 Chip

- 胶囊形；
- 可多选；
- 选中态使用蓝色浅背景 + 蓝色文字；
- 未选中使用系统灰背景。

### 7.5 卡片

- 20–28pt 圆角；
- 轻阴影；
- 不使用玻璃材质；
- 信息层级清楚；
- 支持文字换行。

### 7.6 图表

- 简洁折线；
- 有数据日期显示点；
- 无数据日期不显示点；
- 视觉连接有效点；
- 提供文本摘要；
- 支持 VoiceOver 描述。

## 9. App Store 与合规风险提醒

### 9.1 健康数据

必须说明：仅供一般健康参考，不构成医疗诊断，不替代医生、营养师、心理咨询师建议。

### 9.2 HealthKit

必须：

- 明确读写用途；
- 读取和写入分别授权；
- 仅请求体重相关必要权限；
- 说明用户可以随时关闭；
- 不把 Apple Health 数据用于广告追踪。

### 9.3 AI

必须：

- 标识为鼓励反馈，不是医疗建议；
- 不给药物建议；
- 不鼓励极端节食；
- 用户关闭 AI 后停止发送上下文；
- 服务端日志最多 14 天。

### 9.4 社区附录模块

启用社区前必须补：内容审核、举报、拉黑、评论治理、未成年人保护、图片隐私、注销物理删除规则。

## 12. 结论

微轻的页面设计应坚持：

```text
轻量记录，而不是重型健康管理；
温柔陪伴，而不是焦虑驱动；
苹果原生，而不是过度装饰；
隐私优先，而不是炫耀数字；
清晰可用，而不是只追求好看。
```

在此基础上，后续设计稿生成、Figma 组件化、SwiftUI 实现都应围绕同一套 HIG 合规约束推进。
